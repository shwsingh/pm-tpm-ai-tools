🟡 UI Modules

The nine Streamlit pages a TPM interacts with directly — one per capability built across Days 1–12.

🚦Day 2
Launch Risk Analyzer ↗
Role
Accepts a launch description; runs keyword-based risk scoring across High / Medium / Low tiers; returns an exec-ready risk summary.
Impact
Gives TPMs an instant risk read on any launch without an LLM call — fast, explainable, zero API cost.
🐛Day 5
Bug Triage Agent ↗
Role
Takes a raw bug report; Claude classifies severity (P0–P3), assigns an owner team, and decides whether to escalate.
Impact
Cuts triage time from minutes to seconds; consistent severity calls across the team.
🔁Day 6
Agent Workflow ↗
Role
3-stage pipeline UI: Ingest → Triage → Escalation Handler. Code controls the sequence; session state passes data between stages.
Impact
Demonstrates deterministic multi-step orchestration — the contrast that makes Day 12's agent loop meaningful.
📊Day 7
Status Report ↗
Role
Auto-populates an exec status report from pipeline session state; runs a 5-dimension quality eval; human confirms before finalizing.
Impact
Removes the blank-page problem for weekly updates; quality eval catches weak reports before they reach executives.
📚Day 8
Knowledge Base ↗
Role
Upload .txt, .md, .docx, .csv files; stores chunked text in session state; keyword search returns matching passages.
Impact
Turns unstructured docs into a searchable context layer — the foundation for RAG in later days.
💬Day 9
Feedback Agent ↗
Role
Claude returns per-item sentiment / themes / severity / action plus aggregate top themes and TPM next steps.
Impact
Surfaces what customers actually care about in seconds; replaces manual tagging of feedback spreadsheets.
🕸️Day 10
Dependency Agent ↗
Role
Structured form to add cross-team dependencies; Claude reasons over the graph to identify critical path, cascade risks, and TPM actions.
Impact
Makes hidden cross-team risk visible before it becomes a slip; gives TPMs a defensible escalation narrative.
🧪Day 11
Eval Framework ↗
Role
Scores any skill output against the skill spec's evaluation rules via a second Claude call (Claude-as-judge).
Impact
Makes quality measurable and repeatable; enables regression testing when prompts or models change.
🧠Day 12
Orchestrator ↗
Role
Free-text TPM request → agent loop → Claude picks tools → tools execute → Claude synthesizes an executive briefing.
Impact
The only surface where Claude drives its own reasoning trajectory — genuine multi-agent orchestration vs. scripted pipelines.

🟢 Agent Infrastructure

The plumbing that connects every Claude call — retry logic, token logging, and the loop pattern.

⚙️
AgentHarness ↗
Role
Single gateway for every Claude call — system prompt injection, JSON-parse retry (up to 2×), token logging, session state recording.
Impact
Every Claude call is reproducible, observable, and evaluable. Token costs visible per call and cumulative across the session.
🪜
3-Stage Pipeline ↗
Role
Ingest → Triage → Escalation sequence. Code controls the flow; each stage passes its output as input to the next via session state.
Impact
Shows "code orchestrates Claude" — the deliberate contrast to the Day 12 loop where Claude orchestrates itself.
🔄
Agent Loop ↗
Role
Tool-use agentic loop: call Claude → check stop_reason → if tool_use, dispatch and feed results back → repeat until end_turn or 5 iterations.
Impact
Enables Claude to reason across multiple tool calls autonomously — foundation for Day 13 MCP and Day 14 full copilot.

🔵 Skill Specs

Markdown prompt contracts — input/output schemas, evaluation rules, and upgrade paths. Not code; consumed by agents.

📄
skill: launch_risk_analysis ↗
Role
Input/output contract, JSON schema, evaluation rules, and expected output example for the Launch Risk Analyzer.
Impact
Decouples prompt logic from app code; reusable across any LLM or surface.
📄
skill: bug_triage ↗
Role
One-bug-per-call rule, severity / owner / escalation output schema, and eval criteria for the triage agent.
Impact
Makes the triage contract explicit — prevents batching bugs, a known failure mode where Claude merges severities.
📄
skill: status_report ↗
Role
5 quality dimensions (completeness, clarity, risk visibility, action orientation, exec tone) and scoring rubric.
Impact
The rubric is what makes Claude-as-judge produce meaningful scores vs. generic positive feedback.
📄
skill: knowledge_base ↗
Role
Chunking strategy, keyword match logic, and retrieval output format for the Knowledge Base.
Impact
Defines the retrieval contract so the KB can be swapped to vector search without changing callers.
📄
skill: feedback_agent ↗
Role
Per-item + aggregate output schema, sentiment labels, severity thresholds, and TPM action format.
Impact
Structured output means feedback results can flow directly into downstream tools (Jira, Slack) without parsing.
📄
skill: dependency_agent ↗
Role
Graph input schema, cascade detection reasoning pattern, critical path output format, and MCP upgrade path.
Impact
Explicitly documents the Day 13 upgrade path — MCP replaces the form input with live Jira/GitHub data.

🟣 Data & LLM

Where state lives and how Claude is invoked — the runtime layer that connects every module.

🤖
Claude API ↗
Role
All calls route through AgentHarness. Day 12 also calls it directly inside the agent loop for tool-use decisions.
Impact
Single intelligence layer; swapping models only requires changing the MODEL constant in one place.
💾
Session State ↗
Role
Streamlit in-memory bus passing data between pipeline stages. Cleared on page reload.
Impact
Zero infrastructure cost. The tradeoff — data resets on reload — is why Day 13 MCP replaces it with persistent context.
📑
Doc Chunks ↗
Role
Text passages extracted from uploaded KB files; stored in session state; searched by keyword at query time.
Impact
Enables context-grounded answers without a vector DB. The chunking approach is the bottleneck RAG will fix.
🕸️
Dependency Graph ↗
Role
List of dependency dicts accumulated via the structured form; serialized to JSON and sent to Claude for graph reasoning.
Impact
Structured representation keeps cascade reasoning grounded — Claude can't hallucinate deps that weren't entered.
📈
Token Log ↗
Role
Running list of meta dicts from every AgentHarness call; powers the Token Usage Dashboard at the bottom of the app.
Impact
Makes cost visible in real time; the data needed to compare model and prompt efficiency across days.

⬛ Git Infrastructure

The automation layer that keeps documentation in sync with code across every day of the challenge.

📝
Git Commits ↗
Role
Day N — ... commit messages mark each day's milestone and trigger the post-commit hook automatically.
Impact
Naming convention doubles as machine-readable signal — no extra config or tooling required.
🪝
post-commit Hook ↗
Role
Fires after every Day N commit; checks that progress_tracker.md, 14_day_plan.md, README.md, and a design_decisions/dayN file were all updated.
Impact
Catches documentation drift at commit time — the earliest possible feedback loop, before the next session starts.
🗂️
Tracking Files ↗
Role
progress_tracker.md (status + lessons), 14_day_plan.md (✅ done markers), README.md (badge, table, diagram) — the three files the hook enforces.
Impact
Single source of truth for progress; stays in sync with code because the hook makes skipping it friction.
